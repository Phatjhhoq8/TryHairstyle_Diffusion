\section{Môi trường và Cài đặt hệ thống}
Để đáp ứng ngưỡng hiệu năng huấn luyện và xử lý suy luận khổng lồ từ Stable Diffusion XL, hệ thống yêu cầu nền tảng kỹ thuật tiên tiến:
\begin{itemize}
    \item \textbf{Phần cứng đồ họa:} Tối thiểu NVIDIA RTX 3060 với \textbf{12GB VRAM}. Nền tảng thực thi ép kiểu điểm ảnh dấy phẩy động \textbf{Half Tensor (FP16)} thông qua chế độ CUDA AutoCast nhằm tiết kiệm hơn 40\% dung lượng tải bộ nhớ.
    \item \textbf{Môi trường lõi:} Ứng dụng hoạt động song song qua hệ thống Linux ảo hóa (WSL2) hoặc Container Docker. Các phụ thuộc thư viện (\textit{Dependencies}) Python phải giới hạn tính chính xác ở \texttt{transformers==4.49.0} để giải quyết xung đột MT5Tokenizer với Diffusers lõi.
\end{itemize}

\section{Xây dựng Model AI (Giai đoạn Huấn luyện)}
Chất lượng sinh trưởng tóc của mô hình phụ thuộc vào kiến trúc \textbf{Two-Stage Training} phân cấp hoàn toàn so với mô-típ fine-tuning truyền thống.

\subsection{Giai đoạn 1: Khởi tạo Bộ mã hóa kết cấu (Texture Encoder)}
Khối mã hóa \texttt{HairTextureEncoder} xây dựng trên gốc ImageNet \textbf{ResNet-50}. File thực thi loại bỏ các khối FC gốc để chuyển đổi hình thái mảnh tóc phân loại $128 \times 128$ pixel sang vector $2048$-chiều. Quá trình kiểm soát độ lệch chu kỳ (loss) bằng định lý \textbf{Supervised Contrastive Loss} ($\text{Temperature} = 0.07$) phối hợp \textit{log-sum-exp trick}; hệ quả là biểu đồ dải vector học hút các mô tả nếp tóc giống nhau vào chung một vị trí và đẩy các nhóm tóc khác kiểu ra xa biên tương đồng.

\subsection{Giai đoạn 2: Tối ưu mạng UNet Inpainting bằng LoRA}
Toàn bộ mạng SDXL được "đóng băng" tĩnh 99,7\% (bảo vệ 2,6 tỷ tham số), hệ thống thiết lập \textbf{Low-Rank Adaptation (LoRA)} với thứ hạng ma trận r=16 (chỉ train $\sim0,3\%$ mô hình trọng số).
Các thuật toán tối ưu hội tụ (Loss convergence) mang tính cách mạng được nạp vào GPU bao gồm:
\begin{itemize}
    \item \textbf{Edge-weighted MaskAwareLoss:} Vùng mạng nơ-ron tái tạo chệch choạc tại chân tóc phạt tỷ lệ lỗi trọng số độ lệch gấp $3.0$ lần (trọng số cao nhất). Vùng giữa mảng tóc đánh giá $1.0$, vùng triệt tiêu rỗng phạt $0.1$. Yếu tố này rèn luyện Unet khả năng viền nét không lem mép mảnh.
    \item \textbf{Latent Perceptual Loss (L1):} Đo lường sai số ma trận L1 điểm ảnh tóc hoàn toàn tại tầng mã hóa tiềm ẩn \textit{Latent Space}, bỏ qua hoàn toàn VAE Decode (giảm quá tải VRAM tới 2GB lúc huấn luyện).
    \item \textbf{Tăng tốc Min-SNR Weighting:} Điều hướng nhạy sáng học mô hình ở các tệp timestep quan trọng $\dots$ tốc độ hội tụ mô hình tăng $30\%$.
\end{itemize}
Việc tích hợp \textbf{CFG Dropout 10\%} ngẫu nhiên thả các dữ liệu đầu vào ngữ cảnh thành chuỗi zeros ép UNet không được sụp đổ lười biếng mà phải tự phân tích cấu trúc nền ảnh rỗng, tuân thủ Classifier-Free Guidance.

\section{Xây dựng Phân hệ chuyển đổi Màu (Hair Color Service)}
Dù mô hình Stable Diffusion có khả năng đổi tông màu sinh ra nhờ \textit{Text Prompt}, quá trình này gây cản trở và sửa đổi bản sắc luồng dữ liệu kiến trúc lọn tóc. Gói \texttt{Hair Color Service} ra đời hoạt động song song.
Hệ thống tận dụng chuyển vị đồ họa điểm ảnh (Pixel Space) bằng phân nhánh đổi hệ không gian \textbf{HSV Color Transfer}. Kênh H (Hue - Sắc độ) và S (Saturation - Độ chói) được thế chỗ thành mã màu mới (Vd: Hồng, Bạch kim), hệ lệnh \textbf{giữ cô lập hoàn toàn} ma trận V (Value - Ánh sáng). Cách thức này cấp cho người dùng một mái tóc màu mới lập tức sau 1 giây nhưng không làm biến mất độ phản chiếu (highlight bóng bẩy) của nếp tóc gốc sinh ra. 

\section{Xây dựng Ứng dụng Giao diện (Frontend \& API)}
Kiến trúc Microservices được phân làm 2 lớp:
\begin{enumerate}
    \item \textbf{FastAPI Backend:} Định nghĩa Swagger UI qua giao thức Pydantic, tiếp nhận HTTP Request kèm Base64, kích hoạt Redis Task Queue và quản trị \texttt{uvicorn} máy chủ. Cây API tích hợp endpoint \texttt{/generate} đổ dòng trạng thái thực tế đến Client.
    \item \textbf{Frontend React/Vite:} Lớp giao diện xây dựng hệ sinh thái TailwindCSS tĩnh tại. Cung cấp chức năng tải lên (Upload) hình ảnh thẻ sinh trắc; bảng tùy chọn tóc phân loại kiểu mẫu. Tiến trình được biểu thị qua Loading Skeleton để tạo cảm giác phản hồi theo thời gian thực (Realtime UX) phù hợp với vòng chờ 5 giây.
\end{enumerate}
